# train.py

Запуск обучения модели YOLOv11n-seg на датасете кошачьих хвостов. После обучения автоматически прогоняет валидацию и выводит метрики.

## Константы

- `MODEL = "yolo11n-seg.pt"` — предобученные на COCO веса (nano-версия, ~2.8M параметров)
- `DATASET = "data/dataset.yaml"` — конфиг датасета (пути к train/val и список классов)

## Параметры обучения

### Основные

| Параметр | Значение | Назначение |
|----------|----------|------------|
| epochs | 150 | Максимум эпох (одна эпоха = модель увидела все изображения один раз) |
| imgsz | 800 | Все фото ресайзятся до 800×800 |
| batch | 8 | Сколько изображений обрабатывается за один шаг обновления весов |
| freeze | 10 | Первые 10 слоёв (backbone) заморожены — обучается только голова |
| patience | 20 | Early stopping: если за 20 эпох нет улучшения — стоп |
| optimizer | AdamW | Алгоритм обновления весов |

### Learning rate

| Параметр | Значение | Назначение |
|----------|----------|------------|
| lr0 | 0.01 | Начальная скорость обучения |
| lrf | 0.01 | Финальный lr = lr0 × lrf = 0.0001 |
| cos_lr | True | Косинусное расписание (плавное снижение lr) |

### Аугментации

| Параметр | Значение | Что делает |
|----------|----------|------------|
| hsv_h | 0.015 | Сдвиг оттенка ±1.5% |
| hsv_s | 0.7 | Сдвиг насыщенности ±70% |
| hsv_v | 0.4 | Сдвиг яркости ±40% |
| degrees | 15 | Поворот ±15° |
| translate | 0.1 | Сдвиг ±10% |
| scale | 0.5 | Масштабирование ±50% |
| fliplr | 0.5 | Горизонтальное отражение (50%) |
| flipud | 0.0 | Вертикальное отражение отключено |
| mosaic | 1.0 | Склейка 4 изображений в одно (100%) |
| mixup | 0.1 | Наложение двух изображений (10%) |
| erasing | 0.1 | Случайное закрашивание области (10%) |
| close_mosaic | 10 | Отключить mosaic за 10 эпох до конца |

## Функции

### `main()`

1. **Загрузка модели** — `YOLO(MODEL)` загружает предобученные веса. Если файла нет — скачивается автоматически
2. **Обучение** — `model.train(...)` запускает обучение со всеми параметрами. Результаты сохраняются в `runs/tail_seg/train_v3*`
3. **Валидация** — после обучения ищет `best.pt` (веса с лучшей метрикой), загружает модель и прогоняет `model.val()` на валидационной выборке
4. **Вывод метрик** — печатает mAP50 и mAP50-95 для box и mask

## Метрики

- **mAP50 (box)** — точность bounding box'ов при IoU ≥ 50%
- **mAP50-95 (box)** — среднее по IoU от 50% до 95% (строже)
- **mAP50 (mask)** — точность масок сегментации при IoU ≥ 50%
- **mAP50-95 (mask)** — самая строгая метрика: точность маски при высоких порогах
