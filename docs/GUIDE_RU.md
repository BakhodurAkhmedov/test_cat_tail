# Пошаговое руководство: Сегментация хвоста кота

Это руководство объясняет каждый шаг простым языком — что делаем, зачем, и как это работает.

---

## Что мы вообще делаем?

Мы учим нейросеть находить на фотографии хвост кота и обводить его контур (это называется **сегментация**). Для этого:

1. Собираем фотографии котов
2. Вручную обводим хвосты на этих фото (разметка)
3. Берём готовую умную нейросеть (YOLO) и дообучаем её на наших данных
4. Делаем простое веб-приложение, куда можно загрузить фото и увидеть результат

## Шаг 1: Сбор изображений

### Что делаем?
Скачиваем 40-50 фотографий котов из интернета.

### Зачем?
Нейросети учатся на примерах. Чем больше разных примеров — тем лучше она будет работать. 40-50 фото — это минимум для нашей задачи.

### Какие фото нужны?
- Хвост должен быть **хорошо виден** (не спрятан за телом)
- **Разные позы**: кот стоит, сидит, идёт, лежит
- **Разные фоны**: дома, на улице, на мебели
- **Разные коты**: рыжие, чёрные, полосатые, пушистые, гладкошёрстные
- **Разное освещение**: дневной свет, лампа, тень

### Откуда брать?
- [Unsplash](https://unsplash.com) — бесплатные качественные фото
- [Pexels](https://pexels.com) — аналогично
- Google Images — но внимание на лицензию

### Скрипт
`scripts/download_images.py` — скачивает фото по URL-ам из текстового файла и приводит их к нужному размеру.

### Результат
Папка `data/raw/` заполнена фотографиями котов.

## Шаг 2: Разметка (Аннотация)

### Что делаем?
Открываем каждую фотографию в программе LabelMe и обводим хвост кота полигоном (многоугольником).

### Зачем?
Нейросеть не знает, что такое "хвост". Мы должны **показать** ей на примерах: "вот на этом фото хвост — вот здесь". Это и есть разметка — мы создаём "правильные ответы", по которым сеть будет учиться.

### Как запустить LabelMe?
```bash
labelme data/raw/ --output data/raw/ --labels tail
```
Откроется графическая программа.

### Как размечать?
1. Открываем фото в LabelMe
2. Нажимаем **"Create Polygons"** (создать полигон)
3. Кликаем мышкой по контуру хвоста — ставим точки вдоль границы хвоста
4. Замыкаем полигон (соединяем последнюю точку с первой)
5. В появившемся окне пишем метку: **tail**
6. Нажимаем **Save**

### Важные правила разметки
- Обводите **только хвост**, не захватывайте тело или лапы
- Начинайте от места, где хвост **отделяется от тела**
- Обводите **плотно по контуру** — не оставляйте большие зазоры
- Включайте **кончик хвоста**
- Если хвост частично скрыт (за стулом, например) — обводите только **видимую часть**
- Ставьте достаточно точек (15-30), чтобы контур был плавным

### Что создаёт LabelMe?
Для каждого фото `cat_01.jpg` появится файл `cat_01.json` с координатами точек полигона. Пример:

```json
{
  "shapes": [
    {
      "label": "tail",
      "points": [[412, 230], [420, 245], [435, 280], ...],
      "shape_type": "polygon"
    }
  ],
  "imageWidth": 1280,
  "imageHeight": 960
}
```

### Результат
В папке `data/raw/` рядом с каждым фото лежит `.json` файл с разметкой.

## Шаг 3: Конвертация разметки в формат YOLO

### Что делаем?
Переводим разметку из формата LabelMe (JSON) в формат YOLO (TXT).

### Зачем?
YOLO не понимает формат LabelMe. У YOLO свой формат: текстовый файл, где каждая строка — это один объект с нормализованными координатами.

### Что значит "нормализованные координаты"?
Вместо пикселей (412, 230) используются доли от размера картинки (0.322, 0.240). Это нужно, чтобы координаты работали при любом размере изображения.

Формула: `x_норм = x_пиксель / ширина_картинки`

### Формат YOLO сегментации
```
<номер_класса> <x1> <y1> <x2> <y2> ... <xN> <yN>
```
Пример:
```
0 0.322 0.240 0.328 0.255 0.340 0.292 ...
```
- `0` — номер класса (у нас один класс "tail", его номер = 0)
- Дальше пары координат (x, y) точек полигона

### Скрипт
`scripts/convert_labelme_to_yolo.py` — читает все `.json` файлы и создаёт `.txt` файлы.

### Результат
Для каждого фото есть `.txt` файл с координатами хвоста в формате YOLO.

## Шаг 4: Разделение на обучающую и проверочную выборки

### Что делаем?
Разделяем все фото на две группы:
- **train** (обучающая) — 80% фото, на них сеть учится
- **val** (валидационная) — 20% фото, на них мы проверяем качество

### Зачем?
Если проверять сеть на тех же фото, на которых она училась — это как давать ученику те же задачи, что были в учебнике. Он может их запомнить, но не понимать. Валидационная выборка — это "контрольная работа" с новыми примерами.

### Скрипт
`scripts/split_dataset.py`

### Результат
```
data/
├── images/
│   ├── train/    # ~32 фото для обучения
│   └── val/      # ~8 фото для проверки
├── labels/
│   ├── train/    # разметка для обучающих фото
│   └── val/      # разметка для проверочных фото
└── dataset.yaml  # конфигурация датасета для YOLO
```

## Шаг 5: Синтетическая аугментация (бонус)

### Что делаем?
Вырезаем хвосты из размеченных фото и вставляем на случайные фоны (пейзажи, текстуры, интерьеры).

### Зачем?
У нас мало данных (40 фото). Аугментация — это способ **искусственно увеличить** датасет. Мы создаём новые комбинации "хвост + фон", которых сеть не видела, и она учится лучше обобщать.

### Как это работает?
1. Берём фото кота с размеченным хвостом
2. По маске (полигону) вырезаем только хвост
3. Применяем случайные трансформации: поворот, масштаб, яркость
4. Вставляем на случайный фон
5. Генерируем новый YOLO-файл разметки для этой картинки

### Какие трансформации применяются?
| Трансформация            | Что делает                          |
|--------------------------|-------------------------------------|
| Поворот                  | Вращает хвост на случайный угол     |
| Горизонтальное отражение | Зеркалит хвост                      |
| Яркость/Контраст        | Делает светлее/темнее               |
| Оттенок/Насыщенность    | Немного меняет цвет                 |
| Размытие                 | Лёгкое гауссово размытие           |

### Скрипт
`scripts/augment_synthetic.py`

### Результат
Ещё 20-30 дополнительных изображений в обучающей выборке.

## Шаг 6: Проверка разметки

### Что делаем?
Рисуем полигоны поверх фотографий, чтобы глазами проверить: правильно ли всё размечено?

### Зачем?
Ошибки в разметке — главный враг качества модели. Если вы случайно обвели лапу вместо хвоста, сеть будет учиться неправильно. Лучше потратить 5 минут на проверку.

### Скрипт
`scripts/verify_labels.py` — создаёт картинки с наложенными масками в папке `data/verify/`.

### Результат
Папка `data/verify/` с фотографиями, на которых хвосты закрашены полупрозрачным цветом. Просмотрите их и убедитесь, что всё правильно.

## Шаг 7: Обучение модели

### Что делаем?
Берём предобученную модель YOLOv11n-seg и дообучаем её находить хвосты котов.

### Что значит "предобученная"?
YOLO уже обучена на огромном датасете COCO (330 000 фото, 80 классов: люди, машины, собаки...). Она уже умеет "видеть" — различать края, текстуры, формы. Нам не нужно учить её с нуля — нужно лишь показать, как выглядят хвосты.

### Что такое "заморозка бэкбона"?
Нейросеть состоит из двух частей:
- **Бэкбон (backbone)** — "глаза" сети, извлекает общие признаки (края, текстуры)
- **Голова (head)** — "мозг" сети, принимает решения (где объект, какой класс)

Мы **замораживаем** бэкбон (не даём ему меняться) и обучаем только голову. Это потому что:
- Бэкбон уже хорошо обучен на COCO
- У нас слишком мало данных, чтобы улучшить бэкбон — скорее испортим
- Обучение только головы быстрее и стабильнее

### Гиперпараметры (настройки обучения)
| Параметр   | Значение | Что это простыми словами                        |
|------------|----------|-------------------------------------------------|
| imgsz      | 640      | Размер картинки для сети (640×640 пикселей)     |
| batch      | 8        | Сколько фото обрабатывается за раз              |
| epochs     | 100      | Сколько раз сеть просмотрит все фото            |
| freeze     | 10       | Заморозить первые 10 слоёв (бэкбон)            |
| patience   | 20       | Остановиться, если 20 эпох нет улучшения        |
| optimizer  | AdamW    | Алгоритм оптимизации (как сеть обновляет веса)  |
| lr0        | 0.01     | Скорость обучения (как быстро сеть учится)      |

### Что такое "эпоха"?
Одна эпоха = один полный проход по всему обучающему набору. Если у нас 32 фото и batch=8, то за эпоху будет 4 шага обучения. За 100 эпох сеть увидит каждое фото 100 раз (каждый раз с разными аугментациями).

### Что такое Early Stopping (patience)?
Если качество на валидации не улучшается 20 эпох подряд — обучение останавливается. Это защита от **переобучения** (когда сеть запоминает обучающие фото, но плохо работает на новых).

### Аугментации при обучении
YOLO автоматически применяет трансформации к каждому фото во время обучения:
- **Поворот** (±15°) — хвосты бывают под разными углами
- **Сдвиг** (±10%) — хвост может быть в любой части кадра
- **Масштаб** (±50%) — коты бывают ближе и дальше от камеры
- **Отражение** — горизонтальное зеркалирование
- **Цвет** — изменение яркости, контраста, оттенка
- **Мозаика** — склеивает 4 фото в одно (больше объектов за шаг)

### Как запустить?
```bash
python train.py
```

### Что происходит во время обучения?
В терминале будут появляться строки вида:
```
Epoch 1/100:  box_loss=1.23  seg_loss=2.45  cls_loss=0.89  mAP50=0.12
Epoch 2/100:  box_loss=1.15  seg_loss=2.30  cls_loss=0.78  mAP50=0.18
...
```

- **box_loss** — ошибка рамки (bounding box) — должна уменьшаться
- **seg_loss** — ошибка сегментации (маски) — должна уменьшаться
- **cls_loss** — ошибка классификации — должна уменьшаться
- **mAP50** — качество на валидации — должно расти

### Метрики качества
| Метрика    | Что измеряет                                       | Хорошо  |
|------------|-----------------------------------------------------|---------|
| mAP50      | Точность при IoU > 50%                              | > 0.55  |
| mAP50-95   | Средняя точность при IoU от 50% до 95%              | > 0.30  |
| Precision  | Сколько найденных хвостов — действительно хвосты    | > 0.70  |
| Recall     | Сколько реальных хвостов модель нашла               | > 0.60  |

**IoU (Intersection over Union)** — мера совпадения предсказанной маски с реальной. IoU=1.0 — идеальное совпадение, IoU=0.5 — совпадение наполовину.

### Результат
- Лучшие веса: `runs/tail_seg/weights/best.pt`
- Графики: `runs/tail_seg/` (кривые потерь, PR-кривая, confusion matrix)

## Шаг 8: Инференс (Предсказание)

### Что делаем?
Запускаем обученную модель на новых фотографиях, которых она не видела.

### Зачем?
Проверяем, что модель работает не только на обучающих фото, но и на новых из интернета. Это главная проверка — модель должна **обобщать**.

### Как запустить?
```bash
# На одном фото
python predict.py --source photo.jpg

# На папке с фото
python predict.py --source test_photos/

# На видео
python predict.py --source video.mp4
```

### Что происходит внутри?
1. Модель загружает фото
2. Масштабирует до 640×640
3. Прогоняет через нейросеть
4. Получает: координаты рамки, маску (полигон), уверенность (confidence)
5. Фильтрует: оставляет только предсказания с confidence > 0.5
6. Рисует на фото: полупрозрачную маску + контур + подпись "tail 0.87"

### Результат
Папка `runs/predict/` с обработанными фотографиями.

## Шаг 9: Веб-приложение (Демо)

### Что делаем?
Создаём простую веб-страницу, куда можно загрузить фото и увидеть результат.

### Зачем?
Это удобнее, чем запускать скрипт в терминале. Можно показать результат заказчику/коллегам.

### Как запустить?
```bash
python app/app.py
```
Откроется ссылка (обычно http://localhost:7860). Откройте её в браузере.

### Как пользоваться?
1. Перетащите фото в поле загрузки (или нажмите и выберите файл)
2. Нажмите "Submit"
3. Справа появится фото с наложенной маской хвоста

### Что используется?
**Gradio** — библиотека для быстрого создания веб-интерфейсов. Код приложения занимает ~30 строк.

## Словарь терминов

| Термин                | Объяснение                                                                                      |
|-----------------------|-------------------------------------------------------------------------------------------------|
| **Сегментация**       | Выделение пиксельной маски объекта (не просто рамка, а точный контур)                          |
| **Instance segmentation** | Сегментация каждого отдельного объекта (если два хвоста — две маски)                        |
| **YOLO**              | You Only Look Once — быстрая архитектура нейросети для детекции                                |
| **Transfer Learning** | Дообучение — берём уже обученную сеть и адаптируем под свою задачу                             |
| **Fine-tuning**       | То же, что transfer learning — тонкая настройка модели                                         |
| **Backbone**          | Основная часть сети, извлекает признаки из изображения                                         |
| **Head**              | Верхняя часть сети, принимает решения (класс, рамка, маска)                                    |
| **Epoch**             | Один проход по всему обучающему набору данных                                                  |
| **Batch**             | Группа изображений, обрабатываемых одновременно                                                |
| **mAP**               | Mean Average Precision — основная метрика качества детекции                                     |
| **IoU**               | Intersection over Union — насколько предсказание совпадает с разметкой                         |
| **Overfitting**       | Переобучение — сеть запомнила обучающие фото, но плохо работает на новых                       |
| **Augmentation**      | Аугментация — искусственные преобразования фото для увеличения разнообразия                    |
| **Inference**         | Инференс — применение обученной модели к новым данным                                          |
| **Confidence**        | Уверенность модели (0-1), что она нашла объект                                                 |
| **Polygon**           | Многоугольник — набор точек, описывающих контур объекта                                        |
| **COCO**              | Большой датасет с разметкой (330k фото, 80 классов)                                            |
| **Normalization**     | Приведение координат к диапазону 0-1 (деление на размер картинки)                              |
