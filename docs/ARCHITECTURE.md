# Сегментация хвоста кота — Архитектура

## Обзор

Система инстанс-сегментации кошачьих хвостов на базе YOLOv11n-seg с transfer learning на небольшом датасете (~93 изображения: 43 реальных + 50 синтетических, валидация на 13 изображениях).

## Ключевые решения

### 1. Почему YOLOv11n-seg, а не YOLOv8n-seg?
- Новая архитектура с лучшей точностью при том же размере модели
- Тот же Ultralytics API, замена без изменений кода
- Лучше работает с мелкими объектами (хвосты бывают тонкими)

### 2. Почему LabelMe для разметки?
- Работает локально (данные не уходят в облако)
- Выдаёт полигоны в JSON, которые легко конвертировать в YOLO-формат
- Бесплатный, хорошо документированный, простой

### 3. Почему замораживаем backbone?
- Всего ~90 изображений — обучение всей сети приведёт к переобучению
- Backbone (первые 10 слоёв) предобучен на COCO — уже понимает края, текстуры, формы
- Обучаем только голову (слои детекции и сегментации)

### 4. Почему Gradio, а не Flask/Streamlit?
- Один файл, ~30 строк кода
- Встроенные компоненты для загрузки фото и видео
- Возвращает аннотированное изображение напрямую
- Не нужен HTML/CSS

### 5. Почему два уровня аугментации?
- **Встроенная в YOLO** (в `train.py`): mosaic, mixup, HSV jitter, flip, поворот, erasing — применяется на лету во время обучения
- **Albumentations** (в `augment_synthetic.py`): аугментация вырезанных хвостов перед наложением на фон — создаёт постоянные синтетические изображения на диске

## Поток данных

```
[Сырые фотографии]
    → LabelMe (ручная разметка полигонами)
    → convert_labelme_to_yolo.py (JSON → YOLO .txt)
    → split_dataset.py (80/20 train/val)
    → augment_synthetic.py (наложение хвостов на фон, +50 изображений)
    → verify_labels.py (визуальная проверка)
    → data/images/ + data/labels/ (готово к обучению)
```

## Формат YOLO-разметки

Каждый `.txt` файл содержит одну строку на объект:
```
<class_id> <x1> <y1> <x2> <y2> ... <xN> <yN>
```
- `class_id` = 0 (один класс: `tail`)
- Все координаты нормализованы в диапазон [0, 1] относительно размера изображения
- Вершины полигона определяют маску сегментации

## Пайплайн инференса

```
[Входное изображение/видео]
    → Модель YOLOv11n-seg (предсказание)
    → Постобработка: извлечение масок, скоров уверенности
    → Наложение полупрозрачной маски + контура на оригинал
    → [Выходное аннотированное изображение/видео]
```
