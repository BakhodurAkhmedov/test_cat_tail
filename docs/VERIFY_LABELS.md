# verify_labels.py

Визуальная проверка (QA) разметки. Рисует полупрозрачные зелёные маски поверх изображений и сохраняет результат в `data/verify/`. Позволяет быстро пролистать картинки и увидеть, где разметка неправильная.

## Константы

- `COLOR = (0, 255, 0)` — цвет маски (зелёный, формат BGR)
- `ALPHA = 0.4` — прозрачность маски (0.0 = невидимая, 1.0 = непрозрачная)

## Функции

### `draw_label(img_path, txt_path, out_path)`

Рисует маску на одном изображении.

1. Читает изображение и файл разметки
2. Переводит нормализованные YOLO-координаты обратно в пиксельные
3. Рисует заливку полигона на копии изображения и контур на оригинале
4. Смешивает оба слоя через альфа-блендинг (полупрозрачная маска + чёткий контур)
5. Добавляет текст с количеством хвостов в верхнем левом углу
6. Сохраняет результат

**Возвращает:** `True` при успехе, `False` если изображение не читается.

### `main()`

1. Создаёт папку `data/verify/`
2. Проходит по обеим выборкам (train и val)
3. Для каждого файла разметки ищет соответствующее изображение (пробует `.jpg`, `.jpeg`, `.png`)
4. Вызывает `draw_label` для каждой пары
5. Сохраняет с префиксом выборки: `train_cat_01.jpg`, `val_cat_05.jpg`

## Что позволяет обнаружить

- Полигон не совпадает с хвостом (обвели другой объект)
- Разметка обрезана (обвели только часть хвоста)
- Ошибка конвертации координат (маска съехала)
